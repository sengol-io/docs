The Reports API gives you programmatic access to the full assessment report that Sengol generates for each completed evaluation. You can request either a PDF export — suitable for sharing with stakeholders and regulators — or an Excel export for importing risk data into your own risk registry or compliance tooling. The endpoint returns a pre-signed download URL that you can use directly with any HTTP client.

<Warning>
  Download URLs expire 15 minutes after they are issued. If your URL has expired, make a new request to this endpoint to receive a fresh one.
</Warning>

---

## GET /reports/{assessmentId}

Get a pre-signed download URL for the report of a completed assessment. You must specify the desired file format using the `format` query parameter.

<Note>
  PDF and Excel report exports require a Growth or Enterprise plan with an API add-on. Calling this endpoint without the required plan returns a `403 insufficient_permissions` error. See [Plans and Pricing](/plans-and-pricing) for details.
</Note>

<CodeGroup>
```bash curl
curl "https://api.sengol.ai/v1/reports/asmt_xyz789?format=pdf" \
  -H "Authorization: Bearer sk_live_YOUR_API_KEY"
```

```python Python
import os
import requests

response = requests.get(
    "https://api.sengol.ai/v1/reports/asmt_xyz789",
    params={"format": "pdf"},
    headers={"Authorization": f"Bearer {os.environ['SENGOL_API_KEY']}"},
)
print(response.json())
```

```javascript Node.js
const response = await fetch(
  "https://api.sengol.ai/v1/reports/asmt_xyz789?format=pdf",
  {
    headers: { Authorization: `Bearer ${process.env.SENGOL_API_KEY}` },
  }
);
const data = await response.json();
console.log(data);
```
</CodeGroup>

### Query Parameters

<ParamField query="format" type="string" required>
  The file format for the report. Accepted values are `pdf` and `excel`.
</ParamField>

### Response

<ResponseField name="url" type="string">
  A pre-signed HTTPS URL pointing to the report file in Sengol's secure storage. The URL does not require an `Authorization` header — use it directly with any HTTP client or redirect a browser to it.
</ResponseField>

<ResponseField name="expiresAt" type="string">
  ISO 8601 timestamp indicating when the pre-signed URL expires. URLs are valid for 15 minutes from the time of issue.
</ResponseField>

<ResponseField name="format" type="string">
  The format of the report, echoing your request parameter. Either `pdf` or `excel`.
</ResponseField>

```json
{
  "data": {
    "url": "https://storage.sengol.ai/reports/asmt_xyz789_report.pdf?X-Amz-Expires=900&X-Amz-Signature=...",
    "expiresAt": "2025-01-15T11:00:00Z",
    "format": "pdf"
  },
  "meta": {
    "requestId": "req_01HX9MZMQ3BFPV2NTW8YCEJ6E",
    "timestamp": "2025-01-15T10:45:00Z"
  }
}
```

## Downloading the Report

Once you have the pre-signed URL from the response, use it to download the file with any HTTP client. The URL requires no additional authentication headers.

<CodeGroup>
```bash curl
# Download the PDF report to the current directory
curl -o customer_churn_predictor_report.pdf \
  "https://storage.sengol.ai/reports/asmt_xyz789_report.pdf?X-Amz-Expires=900&X-Amz-Signature=..."
```

```python Python
import requests

# Use the URL from the previous GET /reports response
download_url = "https://storage.sengol.ai/reports/asmt_xyz789_report.pdf?..."

response = requests.get(download_url)
with open("customer_churn_predictor_report.pdf", "wb") as f:
    f.write(response.content)
print("Report saved.")
```

```javascript Node.js
import fs from "fs";

// Use the URL from the previous GET /reports response
const downloadUrl = "https://storage.sengol.ai/reports/asmt_xyz789_report.pdf?...";

const response = await fetch(downloadUrl);
const buffer = await response.arrayBuffer();
fs.writeFileSync("customer_churn_predictor_report.pdf", Buffer.from(buffer));
console.log("Report saved.");
```
</CodeGroup>

## Report Contents

The content and structure of the exported report depends on the format you request.

<Tabs>
  <Tab title="PDF Report">
    The PDF report is formatted for executive and regulatory audiences. It contains the following sections:

    - **Executive summary** — one-page overview of the AI system, overall risk score, and key findings
    - **Category breakdown** — detailed scores and commentary for Performance, Security, Compliance, and Operational dimensions
    - **Top 5 risks** — each risk listed with severity, description, and recommended mitigation action
    - **Cost estimates** — estimated remediation costs for each identified risk
    - **Evidence insights** — a curated selection of matched real-world AI incidents that informed the assessment
    - **Compliance framework mapping** — a table showing how the system maps to applicable frameworks (EU AI Act, NIST AI RMF, ISO 42001, GDPR) based on declared jurisdictions
    - **Assessment metadata** — system description, assessment date, jurisdiction, and report generation timestamp
  </Tab>
  <Tab title="Excel Report">
    The Excel report structures all assessment data into tabular worksheets, designed for import into risk registries and further quantitative analysis. It includes:

    - **Summary sheet** — overall and category scores in a single-row format ready for aggregation
    - **Risks sheet** — all identified risks with severity, description, recommendation, and cost estimate as separate columns
    - **Evidence sheet** — all matched incidents with relevance scores and match factors
    - **Compliance mapping sheet** — framework-by-framework compliance gap analysis in tabular form
    - **Metadata sheet** — system profile fields and assessment configuration

    <Tip>
      The Excel format is ideal for teams that maintain a centralised AI risk registry and want to import Sengol findings directly into their own tooling without manual re-entry.
    </Tip>
  </Tab>
</Tabs>
