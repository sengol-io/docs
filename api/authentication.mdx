The Sengol API authenticates requests using API keys passed as Bearer tokens. Every request you make must include a valid API key in the `Authorization` header — requests without a key, or with an invalid one, are rejected with a `401` response.

<Warning>
  API keys provide full access to your account data. Treat them like passwords and never share them publicly.
</Warning>

## Generating an API Key

<Steps>
  <Step title="Log in to your dashboard">
    Go to [sengol.ai/dashboard](https://sengol.ai/dashboard) and sign in to your account.
  </Step>
  <Step title="Open API Keys settings">
    Navigate to **Account Settings** → **API Keys** from the left sidebar.
  </Step>
  <Step title="Create a new key">
    Click **Generate New Key**, enter a descriptive name (for example, `production-ci` or `dev-local`), and confirm.
  </Step>
  <Step title="Copy and store the key">
    Copy the key immediately and store it in a secure location such as a secrets manager. Sengol will not display the full key again after you leave this page.
  </Step>
</Steps>

## Using Your API Key

Include your API key as a Bearer token in the `Authorization` header on every request:

```
Authorization: Bearer sk_live_YOUR_API_KEY
```

The example below shows a complete authenticated request using curl:

<CodeGroup>
```bash curl
curl https://api.sengol.ai/v1/assessments \
  -H "Authorization: Bearer sk_live_YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

```python Python
import os
import requests

response = requests.get(
    "https://api.sengol.ai/v1/assessments",
    headers={
        "Authorization": f"Bearer {os.environ['SENGOL_API_KEY']}",
        "Content-Type": "application/json",
    },
)
print(response.json())
```

```javascript Node.js
const response = await fetch("https://api.sengol.ai/v1/assessments", {
  headers: {
    Authorization: `Bearer ${process.env.SENGOL_API_KEY}`,
    "Content-Type": "application/json",
  },
});
const data = await response.json();
console.log(data);
```
</CodeGroup>

## API Key Security Best Practices

Following these practices reduces the risk of accidental key exposure and minimises the blast radius if a key is ever compromised.

- **Never commit API keys to source control.** Even in private repositories, secrets committed to Git are difficult to fully purge and create ongoing risk.
- **Use environment variables.** Reference your key as `SENGOL_API_KEY` (or equivalent) and inject it at runtime via your CI/CD system or a secrets manager such as AWS Secrets Manager, HashiCorp Vault, or Doppler.
- **Rotate keys regularly.** Treat key rotation as routine hygiene — schedule it quarterly or immediately after any suspected exposure.
- **Use separate keys per environment.** Maintain distinct keys for development, staging, and production so you can revoke a compromised key without affecting other environments.

<Tip>
  Name your keys clearly (for example, `github-actions-prod`) so you can identify and revoke them quickly if needed. You can create as many keys as your plan allows.
</Tip>

## Authentication Errors

| HTTP Status | Code | Cause | Resolution |
|---|---|---|---|
| 401 | `invalid_api_key` | The key is incorrect, malformed, or has been revoked | Regenerate a new key in Account Settings |
| 401 | `missing_api_key` | No `Authorization` header was included in the request | Add the `Authorization: Bearer <key>` header |
| 403 | `insufficient_permissions` | Your plan does not include API access | Upgrade to a plan with an API add-on at [sengol.ai/contact](https://sengol.ai/contact) |
