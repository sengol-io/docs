The Assessments API is the core of the Sengol platform. You use it to submit an AI system for analysis, poll for results as Sengol processes the assessment, and retrieve the completed risk evaluation. Assessments are associated with a project for organisation but can also be created independently.

<Note>
  Assessment processing is asynchronous. Poll `GET /assessments/{id}` every 5 seconds until the status changes to `completed` before attempting to retrieve risk scores or download reports.
</Note>

---

## POST /assessments

Start a new risk assessment for an AI system. Sengol queues the assessment immediately and returns a pending assessment object. Processing typically completes within 30–60 seconds.

<CodeGroup>
```bash curl
curl -X POST https://api.sengol.ai/v1/assessments \
  -H "Authorization: Bearer sk_live_YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "projectId": "proj_abc123",
    "system": {
      "name": "Customer Churn Predictor",
      "industry": "financial_services",
      "useCase": "decision_automation",
      "dataTypes": ["pii", "financial"],
      "userScale": 50000,
      "jurisdiction": ["US", "EU"]
    }
  }'
```

```python Python
import os
import requests

payload = {
    "projectId": "proj_abc123",
    "system": {
        "name": "Customer Churn Predictor",
        "industry": "financial_services",
        "useCase": "decision_automation",
        "dataTypes": ["pii", "financial"],
        "userScale": 50000,
        "jurisdiction": ["US", "EU"],
    },
}

response = requests.post(
    "https://api.sengol.ai/v1/assessments",
    json=payload,
    headers={"Authorization": f"Bearer {os.environ['SENGOL_API_KEY']}"},
)
print(response.json())
```

```javascript Node.js
const response = await fetch("https://api.sengol.ai/v1/assessments", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.SENGOL_API_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    projectId: "proj_abc123",
    system: {
      name: "Customer Churn Predictor",
      industry: "financial_services",
      useCase: "decision_automation",
      dataTypes: ["pii", "financial"],
      userScale: 50000,
      jurisdiction: ["US", "EU"],
    },
  }),
});
const data = await response.json();
console.log(data);
```
</CodeGroup>

### Request Body

<ParamField body="projectId" type="string">
  The ID of the project to associate this assessment with. If omitted, the assessment is created without a project.
</ParamField>

<ParamField body="system" type="object" required>
  Describes the AI system being assessed.

  <Expandable title="system properties">
    <ParamField body="system.name" type="string" required>
      A human-readable name for the AI system (for example, `"Customer Churn Predictor"`).
    </ParamField>

    <ParamField body="system.industry" type="string" required>
      The industry sector in which the system operates. Accepted values include `financial_services`, `healthcare`, `retail`, `insurance`, `legal`, `education`, and `government`.
    </ParamField>

    <ParamField body="system.useCase" type="string" required>
      The primary use case for the system. Accepted values include `decision_automation`, `content_generation`, `recommendation`, `classification`, `forecasting`, and `natural_language_processing`.
    </ParamField>

    <ParamField body="system.dataTypes" type="string[]">
      An array of data types the system processes. Accepted values include `pii`, `financial`, `health`, `biometric`, `behavioural`, and `public`.
    </ParamField>

    <ParamField body="system.userScale" type="integer">
      The approximate number of users or transactions the system handles per day. Used to calibrate operational risk scoring.
    </ParamField>

    <ParamField body="system.jurisdiction" type="string[]">
      An array of jurisdictions where the system will be deployed (for example, `["US", "EU", "UK"]`). Used to determine applicable compliance frameworks.
    </ParamField>
  </Expandable>
</ParamField>

## GET /assessments/{id}

Retrieve the current state and results of an assessment by its ID. Once the status is `completed`, the response includes the top-level `scores` summary. For the full per-category breakdown, use [GET /risk-scores/{assessmentId}](/api/risk-scores).

<CodeGroup>
```bash curl
curl https://api.sengol.ai/v1/assessments/asmt_xyz789 \
  -H "Authorization: Bearer sk_live_YOUR_API_KEY"
```

```python Python
import os
import requests

response = requests.get(
    "https://api.sengol.ai/v1/assessments/asmt_xyz789",
    headers={"Authorization": f"Bearer {os.environ['SENGOL_API_KEY']}"},
)
print(response.json())
```

```javascript Node.js
const response = await fetch(
  "https://api.sengol.ai/v1/assessments/asmt_xyz789",
  {
    headers: { Authorization: `Bearer ${process.env.SENGOL_API_KEY}` },
  }
);
const data = await response.json();
console.log(data);
```
</CodeGroup>

### Response Fields

<ResponseField name="id" type="string">
  Unique identifier for the assessment.
</ResponseField>

<ResponseField name="status" type="string">
  Current processing status. One of `pending`, `processing`, `completed`, or `failed`.
</ResponseField>

<ResponseField name="system" type="object">
  The AI system metadata submitted when the assessment was created.
</ResponseField>

<ResponseField name="projectId" type="string">
  The project this assessment belongs to, if any.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of when the assessment was created.
</ResponseField>

<ResponseField name="completedAt" type="string">
  ISO 8601 timestamp of when processing finished. Present only when status is `completed` or `failed`.
</ResponseField>

<ResponseField name="scores" type="object">
  High-level score summary. Present only when status is `completed`. Use [GET /risk-scores/{assessmentId}](/api/risk-scores) to retrieve the full breakdown.

  <Expandable title="scores properties">
    <ResponseField name="scores.overall" type="number">
      Composite risk score from 0 to 100. Higher scores indicate lower risk.
    </ResponseField>

    <ResponseField name="scores.performance" type="number">
      Performance dimension score (0–100).
    </ResponseField>

    <ResponseField name="scores.security" type="number">
      Security dimension score (0–100).
    </ResponseField>

    <ResponseField name="scores.compliance" type="number">
      Compliance dimension score (0–100).
    </ResponseField>

    <ResponseField name="scores.operational" type="number">
      Operational dimension score (0–100).
    </ResponseField>
  </Expandable>
</ResponseField>

<Tip>
  Assessments typically move from `pending` → `processing` → `completed` within 30–60 seconds. If an assessment reaches `failed` status, check the `error` field in the response for details and retry the request.
</Tip>

## GET /assessments

List all assessments in your account, ordered by creation date descending. Use query parameters to filter and paginate results.

<CodeGroup>
```bash curl
curl "https://api.sengol.ai/v1/assessments?projectId=proj_abc123&limit=20&offset=0" \
  -H "Authorization: Bearer sk_live_YOUR_API_KEY"
```

```python Python
import os
import requests

response = requests.get(
    "https://api.sengol.ai/v1/assessments",
    params={"projectId": "proj_abc123", "limit": 20, "offset": 0},
    headers={"Authorization": f"Bearer {os.environ['SENGOL_API_KEY']}"},
)
print(response.json())
```

```javascript Node.js
const params = new URLSearchParams({ projectId: "proj_abc123", limit: 20, offset: 0 });
const response = await fetch(
  `https://api.sengol.ai/v1/assessments?${params}`,
  {
    headers: { Authorization: `Bearer ${process.env.SENGOL_API_KEY}` },
  }
);
const data = await response.json();
console.log(data);
```
</CodeGroup>

### Query Parameters

<ParamField query="projectId" type="string">
  Filter results to assessments belonging to a specific project.
</ParamField>

<ParamField query="status" type="string">
  Filter by assessment status. One of `pending`, `processing`, `completed`, or `failed`.
</ParamField>

<ParamField query="limit" type="integer">
  Number of assessments to return per page. Defaults to `20`, maximum `100`.
</ParamField>

<ParamField query="offset" type="integer">
  Number of assessments to skip for pagination. Defaults to `0`.
</ParamField>
